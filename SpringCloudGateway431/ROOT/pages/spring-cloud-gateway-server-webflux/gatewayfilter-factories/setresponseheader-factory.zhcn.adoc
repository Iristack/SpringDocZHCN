[[setresponseheader-gatewayfilter-factory]]
= `SetResponseHeader` `GatewayFilter` 工厂

`SetResponseHeader` `GatewayFilter` 工厂接收 `name` 和 `value` 两个参数。  
以下配置示例定义了一个 `SetResponseHeader` `GatewayFilter`：

.application.yml
[source,yaml]
----
spring:
  cloud:
    gateway:
      routes:
      - id: setresponseheader_route
        uri: https://example.org
        filters:
        - SetResponseHeader=X-Response-Red, Blue
----

该 GatewayFilter 会**替换**（而非添加）所有具有指定名称的响应头。  
例如，如果下游服务器返回了 `X-Response-Red:1234`，它将被替换为 `X-Response-Red:Blue`，网关客户端最终收到的就是这个值。

`SetResponseHeader` 能够识别用于匹配路径或主机的 URI 变量。  
这些变量可以在值中使用，并在运行时动态展开。  
以下示例配置了一个使用变量的 `SetResponseHeader` `GatewayFilter`：

.application.yml
[source,yaml]
----
spring:
  cloud:
    gateway:
      routes:
      - id: setresponseheader_route
        uri: https://example.org
        predicates:
        - Host: {segment}.myhost.org
        filters:
        - SetResponseHeader=foo, bar-{segment}
----
